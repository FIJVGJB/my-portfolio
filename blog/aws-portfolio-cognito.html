Building a Secure AWS Portfolio & Cognito Admin

How I built, secured, and debugged my AWS-hosted portfolio and admin page.

Introduction

This blog details the process of building my portfolio website on AWS S3, integrating CloudFront, Route 53, and securing the admin page with AWS Cognito. I share configuration steps, debugging challenges, and lessons learned, including a real-world billing issue with ACLs.

Portfolio Website on AWS

  • Hosted static site on S3 with CloudFront CDN
  • Custom domain setup via Route 53
  • SSL certificate for secure HTTPS
  • Automated CI/CD with GitHub Actions

One challenge was DNS propagation delays, which I resolved by verifying records and using AWS support. Another was an accidental ACL charge, which I fixed by removing unnecessary ACLs and requesting a refund.

Securing the Admin Page with Cognito

  • Configured AWS Cognito user pool
  • Restricted admin access to authorized users only
  • Tested access control with multiple user roles

Initially, the admin page was accessible to everyone. After identifying the misconfiguration, I updated Cognito rules to restrict access, ensuring only authorized users could log in.

Lessons Learned

  • Always double-check AWS permissions and billing
  • Automate deployments for reliability
  • Access control is critical for security

Highlight: The accidental ACL charge was a valuable lesson in AWS billing and permissions management. I resolved it quickly and improved my deployment process as a result.
