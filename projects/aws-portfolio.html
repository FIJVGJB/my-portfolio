Portfolio Website on AWS

Hosting a modern portfolio on AWS S3, CloudFront, and Route 53 with secure admin access.

Project Overview

This project involved deploying my personal portfolio website using AWS S3 for static hosting, CloudFront for global CDN, and Route 53 for DNS management. Security was enhanced with AWS Cognito for admin access, and CI/CD was set up for automated deployments.

Key Features

  • Static hosting on S3 with public and private buckets
  • CloudFront distribution for fast global access
  • Route 53 for custom domain and DNS
  • AWS Cognito for secure admin login
  • CI/CD pipeline with GitHub Actions (build, test, deploy)

Challenges & Solutions

  • Route 53 DNS propagation delays – solved by verifying records and using AWS support
  • SSL certificate setup for HTTPS
  • Accidental ACL charge – removed unnecessary ACLs and got refund

Lessons Learned

  • Importance of secure S3 permissions
  • How to automate deployments with GitHub Actions
  • Handling AWS billing and support

Project Details

  • Tools: AWS S3, CloudFront, Route 53, Cognito, GitHub Actions
  • Duration: 1 month
  • Role: Full-stack developer, DevOps
